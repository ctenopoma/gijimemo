# 議事メモ

「人が聞いて殴り書きする」ことに特化した、シンプルな議事録デスクトップアプリです。
Web会議画面を見ながら使えるよう、常に最前面に表示し、操作していない時は半透明になります。
記録した内容は Markdown / テキストでコピーでき、LLM による要約も利用できます。

## 主な機能

| 機能 | 説明 |
|------|------|
| アジェンダカード | 論点ごとにカードを作成。会議中でも任意の場所に追加可能 |
| Markdown / テキスト出力 | ワンクリックでクリップボードにコピー。メールや資料にすぐ貼り付けられる |
| 議事録の保存と検索 | SQLite に蓄積。タイトル・メモ内容を横断する全文検索（FTS5）に対応 |
| LLM 要約 | 任意のエンドポイント（OpenAI 互換 API）を設定してストリーミング要約 |
| 常に最前面 / 半透明 | Web 会議を見ながら使用可能。非アクティブ時に自動で半透明化 |
| 社内プロキシ対応 | LLM リクエストはシステムプロキシをバイパスして直接接続 |

## 技術スタック

- **フロントエンド**: React 18 / Vite / TypeScript / Tailwind CSS / Zustand / Lucide React
- **バックエンド**: Tauri v2 (Rust)
- **データベース**: SQLite（rusqlite、FTS5 による全文検索）
- **HTTP**: reqwest（SSE ストリーミング、プロキシバイパス対応）

## セットアップ

### 必要環境

| ツール | バージョン | 確認コマンド |
|--------|-----------|-------------|
| Node.js | 18 以上 | `node --version` |
| npm | 9 以上 | `npm --version` |
| Rust | stable (1.75+) | `rustc --version` |
| Cargo | 同上 | `cargo --version` |
| WebView2 ランタイム | — | Windows 10/11 は通常インストール済み |

> **macOS** の場合は Xcode Command Line Tools が必要です。
> **Linux** の場合は `libgtk-3-dev`, `libwebkit2gtk-4.1-dev` 等が必要です。

### インストール

```bash
git clone <repo-url>
cd gijimemo
npm install
```

### 開発サーバー起動

```bash
npm run tauri dev
```

### プロダクションビルド

```bash
npm run tauri build
```

ビルド成果物は `src-tauri/target/release/bundle/` に生成されます。

## LLM 設定

アプリ右上の **⚙（設定）** アイコンから以下を設定します。

| 項目 | 例 |
|------|----|
| エンドポイント URL | `http://10.0.0.5:8080/v1/chat/completions` |
| API Key | `sk-...`（不要な場合は空欄）|
| モデル名 | `gpt-4o-mini` / `llama3` など |
| プロンプトテンプレート | `{title}`, `{content}` 変数が使用可能 |

**「LLMへの接続確認」** ボタンで疎通確認ができます。

> 社内プロキシ環境の場合も、LLM リクエストは自動的にプロキシをバイパスして直接接続します。
